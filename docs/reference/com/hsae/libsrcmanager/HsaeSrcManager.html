public boolean hasBreakOffState (String type)

Added in API level 2

Description:
check if has out signal that break do src switch

Note:
(1) if the param type set value "null", check whether have any break
(2) if the param type contains "power" and "update". "power": whether in power modes those allow to play or not;"update": whether in USB update now
(3) Check whether have break state which cause source manager cannot give response for hard key and so on.
(4) Check whether have break state which don’t allow to play in HMI.

Parameters
type String: break type， now the type contains "power" and "update"
Returns
boolean true if in break state, otherwise false

public void switchSrc (int app, String pkg, boolean isLoadApp)

Added in API level 2

Description:
try to change source and switch source with startup application
Note:
(1) Source manager will save current source for last mode, so please set "pkg" with valid client package name.
(2) For HMI clients like USB/BT music, local radio, please set "isLoadApp" to "false".
(3) If set "isLoadApp" to "true", source manager will start specified activity by app Id (only music & radio).
(4) If set "isLoadApp" to "false", source manager don’t start any activity.

Parameters
app int: the app mean application Id, the value of application Id refer to HsaeSrcMgrConstants
pkg String: client package name
isLoadApp boolean: flag to identify whether start music/radio activity if top activity is not true: start music/radio activity, false: don't start
Throws
RemoteException remote exception

public void updateInfoToMeter (int app, String obj)

Added in API level 2

Description:
Notify meter and status bar that a media play status,play list and so on from HMI through source manager immediately.

Note:
(1)Parameter “obj” defined in document JSON object
(2)Source manager just pass the JSON string to meter & just give "sub_type", "author" & "title" to status bar to avoid message length over bind’s limit when "sub_type" is "hu_play_start".
(3)If specified app Id (1st parameter) does not match current source Id, update info to meter will be ignored.
(4)Valid values of field “data” & “source” in JSON between DA and IC communication as below:
0: invalid 1: USB music 2: USB video 3: BT music 4: Online music 5: FM 6: AM 7: Online radio

Parameters
app int: the app mean application Id, the value of application Id refer to HsaeSrcMgrConstants
obj String: JSON object ,refer to JSON object
Throws
RemoteException remote exception
